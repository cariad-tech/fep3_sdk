Components

The functional parts of the FEP Participant are called FEP Components. Each FEP Component covers some specific functionality, e. g. functionality to exchange simulation data with other participants.

FEP Components can be accessed via the FEP Participant API’s template method getComponent() in fep3::core::arya::Participant, passing the FEP Component Interface as template argument.

FEP Component System

The FEP Component System provides facilities to make the FEP Participants of a FEP System use other FEP Components than the FEP Default Components, e. g. in order to replace a native component by a custom implementation.

Core API

Core CMake Config

The Core API comes with its own target and can be used as follows:

# find the package
find_package(fep3_participant_core REQUIRED)

# ... define your target and other stuff

# link against the fep3_participant_core
target_link_libraries(my_target PRIVATE
                      fep3_participant_core)

Element Base

See Demo Core Easy Sender and Demo Core Easy Receiver for an example how to use the fep3::core::arya::ElementBase.

The fep3::core::arya::ElementBase is a convenience class that implements the Implementing the fep3::core::arya::ElementBase::load() and fep3::core::arya::ElementBase::initialize(). It handles the type name and the version information of a FEP Element (see fep3::core::arya::ElementBase::getTypename(), fep3::core::arya::ElementBase::getVersion()) and provides access to FEP Components via fep3::core::arya::ElementBase::getComponents(). Calls to fep3::arya::IElement::loadElement() in the context of the FEP Participant’s State Machine are tranferred to calls to fep3::core::arya::ElementBase::load() (fep3::arya::IElement::unloadElement() to fep3::arya::ElementBase::unload() respectively).

Element Configurable

The fep3::core::arya::ElementConfigurable base class provides a convenience class to use the fep3::arya::IConfigurationService.

See Demo Core Easy Sender and Demo Core Easy Receiver for an example how to use the fep3::arya::Configuration.

Easy Logging

The fep3::logging::arya::EasyLogging base class provides a convenience class and macros to use the fep3::arya::ILoggingService. If more customization is required the FEP Logging Service needs to be used, which allows multiple loggers, logger grouping and custom logging sinks.

See Demo Core Easy Sender and Demo Core Easy Receiver for an example how to use the fep3::logging::arya::EasyLogging.

If fep3::core::arya::ElementBase or fep3::core::arya::ElementConfigurable is used then fep3::logging::arya::EasyLogging is already included and does not need to be initialized. To filter the logs from these two classes the logger name “element” has to be used.

Core DataWriter and DataReader

See Demo Core Easy Sender and Demo Core Easy Receiver for an example how to use the core DataWriter and DataReader.

The fep3::core::arya::DataWriter and fep3::core::arya::DataReader are helper classes to add and remove fep3::core::arya::IDataRegistry::IDataReader and fep3::core::arya::IDataRegistry::IDataWriter.

Use the fep3::core::arya::addToComponents() to add Reader or Writer to the Data Registry. Use the fep3::core::arya::removeFromComponents() to remove Reader or Writer from the Data Registry.

DataWriter

DataWriter are queues that will hold the written data samples before they are flushed to the corresponding fep3::arya::IDataRegistry::IDataWriter. The data are written to the bus by calling the fep3::core::arya::DataReader::flushNow(). If the capacity of the DataWriter queue exceeded before it is flushed, the oldest sample will be removed.

The DataWriter can be used to write data with the streaming operator<< for any fep3::core::arya::IDataSample implementation, but also for some other standard and class types. It is designed for the users convenience! If the fep3::core::arya::addToComponents() was used it will automatically set the right timestamp to the data sample. The streaming operator can be used for following data types:

int8_t, uint8_t, int16_t, uint16_t, int32_t, uint32_t, int64_t, uint64_t, float, double

DataReader

DataReader are queues that will hold the reading data samples from the corresponding fep3::arya::IDataRegistry::IDataReader. The reading is triggered by a fep3::core::arya::DataReader::receiveNow() call. This will read the data from the fep3::arya::IDataRegistry::IDataReader only before the given simulation time and keep it in the queue. If the capacity of the DataReader queue exceeded, the oldest sample will be removed.

To read the DataReaders content the streaming operator >> may be used for any fep3::core::arya::IDataSample implementation as well as for some other standard and class types. See table in chapter DataWriter which types are possible.

The streaming operator >> returns the latest sample available. All options to retrieve samples from a data reader are listed in the following table:

Data Reader Access

Description

fep3::core::arya::DataReader::popSampleLatest()

Pops the latest data samples and returns it.

fep3::core::arya::DataReader::popSampleOldest()

Pops the oldest data samples and returns it.

fep3::core::arya::DataReader::readSampleLatest()

Returns the latest data sample without popping it.

fep3::core::arya::DataReader::readSampleOldest()

Returns the oldest data sample without popping it.

fep3::core::arya::DataReader::readSampleBefore()

Returns the oldest data sample with a timestamp below a given upper bound timestamp without popping it.

fep3::core::arya::DataReader::readType()

Returns the current stream type.

Adding Jobs

Jobs are small runnable units to implement a functionality that needs to be part of a distributed scheduler. At the moment jobs configurations are only possible for cyclic time triggered events. In future there might be a possibility to add data triggered jobs as well.

Use the fep3::core::arya::addToComponents() to add a job to the Job Registry. Use the fep3::core::arya::removeFromComponents() to remove the job from the Job Registry.

See Demo Core Easy Sender and Demo Core Easy Receiver for an example to use the core Jobs.

The fep3::arya::IJob::executeDataIn() is to read and prepare the incoming data for the execute call. Do not implement any functional behavior in here, but read all data from the IDataRegistry::IDataReader.

The fep3::arya::IJob::execute() is to implement the jobs functional behavior.

The fep3::arya::IJob::executeDataOut() is to flush the fep3::core::arya::DataWriter queues.

FEP Interface CPP API

FEP Interface CPP CMake Config

The FEP Interface CPP API comes with its own target and can be used as follows:

# find the package
find_package(fep3_participant_cpp REQUIRED)

# ... define your target and other stuff

# link against the fep3_participant_cpp
target_link_libraries(my_target PRIVATE
                      fep3_participant_cpp)

FEP Interface CPP Participant

Use fep3::cpp::arya::createParticipant() to create and use a FEP Participant.

See Demo CPP Easy Sender and Demo CPP Easy Receiver for an example how to use the fep3::cpp::arya::Participant.

DataJob Element

See Demo CPP Easy Sender and Demo CPP Easy Receiver for an example how to use the fep3::cpp::arya::ElementBase.

The fep3::cpp::arya::DataJobElement is a convenience class that implements the Implementing the fep3::core::arya::ElementBase::load() and fep3::core::arya::ElementBase::initialize() and supports one DataJob.

DataJob

The DataJob is a FEP Interface CPP programming model to implement a functionality in an easy way. See Demo CPP Easy Sender and Demo CPP Easy Receiver for an example how to use the fep3::cpp::arya::DataJob.
